The Runtime Manager UI logs on deployment messages and events for the workers that run Mule apps on CloudHub. For applications deployed to your own servers, you can send data to an external tool (such as Splunk or ELK) and then view the data from that tool.

[[navigate_to_runtime_manager_settings]]
include::manage-include-navigate-to-settings.adoc[]

. Click Logs.
+
image:ELMStart.png[ELMStart]
+
. Search for the logs that you want to view.
+
You can search for any exact term in the log message or use common quantifiers for wildcard searching:
+
* `?` - The question mark matches zero or one of the preceding element.
* `*` - The asterisk matches zero or more of the preceding element.

[cols="2,5"]
|===

| Filter pane (shown below): image:ELMfilters.png[ELMfilters]
|
Lets you selectively view the output by day, deployment, system log, and workers.
|===

Note that date and time filters lets you specify a range to search the logs:

* Last hour
* Last 24hrs
* Last week
* Last month

image:ELMAdvancedSearch.png[ELMAdvancedSearch]

You can also search for log content or use advanced search to specify a time or day, date range, or priority.
