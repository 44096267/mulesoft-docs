= To Switch Environments
:keywords: runtime manager, anypoint platform, private cloud

The Runtime Manager UI opens to the default environment where your Mule applications are deployed and executed (as set in Access Management). If you have more than one environment, such as Production and Sandbox environments, you can switch between them to manage the Mule applications they contain.

[[navigate_to_runtime_manager_settings]]
include::manage-include-navigate-to-settings.adoc[]

. Click the name of the environment in the top-left corner to open a window for switching environments.

. Use the Switch Environment dialog that opens to activate a different environment.
