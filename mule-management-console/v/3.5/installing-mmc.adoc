= Installing MMC

MuleSoft offers two installation options for the Mule Management Console: 

[width="100%",cols="50%,50%",]
|===
a|
link:/docs/display/35X/Installing+the+Trial+Version+of+MMC[image:/docs/download/attachments/122750969/MMC-trial.png?version=1&modificationDate=1421450356475[image]]

*link:/docs/display/35X/Installing+the+Trial+Version+of+MMC[Install the trial version] if you are interested in evaluating the Mule Management Console.*

* You can download and install the MMC trial version on your local system without any other technical pre-requisites. 
* The trial version is bundled together with a Mule standalone instance, so you'll be downloading the most recent version of Mule with the MMC agent embedded in it.

 a|
link:/docs/display/35X/Installing+the+Production+Version+of+MMC[image:/docs/download/attachments/122750969/MMC-prod.png?version=1&modificationDate=1421450356178[image]]

*link:/docs/display/35X/Installing+the+Production+Version+of+MMC[Install the production version] if you are planning to use the Mule Management Console in production.*

* You need a valid enterprise license to launch the production version of MMC.
* You need to have the current version of Mule Enterprise already installed.
* The production version of MMC is packaged as a .war file designed to run in an application server or container, such as link:/docs/display/35X/Installing+the+Production+Version+of+MMC#InstallingtheProductionVersionofMMC-tcat[Tcat Server], link:/docs/display/35X/Installing+the+Production+Version+of+MMC#InstallingtheProductionVersionofMMC-tcat[Apache Tomcat], or link:/docs/display/35X/Installing+the+Production+Version+of+MMC#InstallingtheProductionVersionofMMC-jboss[JBoss].

|===
